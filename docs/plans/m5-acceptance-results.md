D-scenario output text:
`科学家们可以得出结论：暗物质对其他暗物质的影响。其他物质的影响方式与普通物质相同。`

LCS vs curated ground truth: **0.780**.

The mechanism gates (one `final` event, segment_count=2, one internal
rotation) all PASS. The text quality FAILs because of a **segment-join
duplication artifact**: the substring `其他物质的影响` is emitted twice,
once at the end of segment 1 (from the carryover audio) and once at the
start of segment 2 (from the same audio re-decoded).

Per design doc §15.5.2 this is exactly the case where the spec calls for
**LCS-based dedup at segment join** ("if duplicate prefixes appear in
measurement, add LCS-based dedup at segment join"). The shipped worker
does not yet dedup. M5 ships verification plumbing (`--long-baseline-text`
+ `--d-lcs-hard-gate`) so the gate can be flipped to a real hard 0.95
gate once the worker-side dedup fix lands.

Full detail: `docs/plans/m3-acceptance-results-zh-long-04.md`.

## Verdict

**P0 release gate PASS.** All three reproduction prompts produce
bit-identical (LCS=1.000) streaming output vs one-shot baseline in both
mel and PCM modes, with end-of-speech latency comfortably inside the
budget.

The long-audio segment-join duplication finding is a known P1 quality
issue documented above; it does not block P0 ship because P0 product
utterances (short voice commands, ≤ ~5 s) never hit auto-segmentation.
